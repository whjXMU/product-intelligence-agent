# 阶段 01.5 报告：项目治理校准

## 阶段目标

在 Agent MVP 子窗口验证成功后，恢复主窗口对项目路线的统一把控，避免实验实现影响正式架构判断。

## 背景

子窗口在 `packages/agent-mvp` 中完成了一次 Agent MVP 能力验证，跑通了手动 HTML 输入、DeepSeek 分析、JSON/Markdown/trace 输出和质量检查。

这是有价值的技术可行性验证，但当前实现仍是 spike，不具备正式产品工程边界。

## 本阶段动作

- 将当前正式阶段校准为“阶段 01.5：项目治理校准与 Agent MVP spike 归档”；
- 明确 `packages/agent-mvp` 是实验包；
- 更新 `agent_share_doc/project-status.md`；
- 更新 `agent_share_doc/handoff.md`；
- 将 `agent_share_doc/stage-reports/02-agent-mvp.md` 定性为 spike 报告；
- 更新 `docs/roadmap.md`；
- 新增 ADR：`0002-agent-mvp-spike-boundary.md`。
- 为 `docs/mvp-subagent-docs/` 增加主窗口校准说明，明确其为参考资料而非正式路线。

## 当前结论

Agent MVP spike 证明了核心智能链路可行，但正式产品路线仍应先补齐业务骨架。

推荐正式路线：

1. 阶段 02：竞品分析任务业务骨架；
2. 阶段 03：正式 Agent Workflow 雏形；
3. 阶段 04：模型 Provider 与结构化输出；
4. 阶段 05：Tools、Guardrails、Observability；
5. 阶段 06：Evaluation 与质量回归。

## 保留资产

- `packages/agent-mvp`：实验包；
- `packages/agent-mvp/output/`：实验输出样例；
- `docs/mvp-subagent-docs/`：子窗口生成的参考资料；
- `agent_share_doc/stage-reports/02-agent-mvp.md`：spike 执行报告。

## 不直接继承

- 不直接把 `packages/agent-mvp` 作为正式 Agent Core；
- 不直接把实验 workflow 接入后端；
- 不直接把实验 prompt 作为正式 prompt 体系；
- 不在正式任务系统前继续扩展复杂 Agent 能力。

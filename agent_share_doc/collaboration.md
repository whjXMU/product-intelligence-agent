# 协作规范

## 角色分工

### 主窗口

负责：

- 总体架构设计；
- 阶段路线规划；
- 关键技术决策；
- 子窗口任务说明；
- 代码复查和验收；
- 维护共享记忆。

### 子窗口

负责：

- 执行边界清晰的阶段任务；
- 遵循既有目录结构和架构约束；
- 完成后更新阶段报告；
- 不擅自引入复杂基础设施。

## 阶段执行模板

每个阶段都遵循：

1. 说明阶段目标；
2. 说明目录变化；
3. 说明关键设计原因；
4. 实际创建或修改文件；
5. 运行验证命令；
6. 总结当前能力、验证方式、下一步。

## 文档约定

- `agent_share_doc/`：跨窗口共享记忆，记录当前状态、交接信息和阶段结果。
- `docs/`：项目正式文档，面向学习、复盘和简历展示。
- `README.md`：项目入口，说明如何启动和验证。

## 决策约定

所有重要架构决策都应写入 `docs/adr/`，包括：

- 为什么使用 monorepo；
- 为什么前后端共享 DTO；
- 为什么当前不引入 RAG、队列、多 Agent；
- 什么时候扩展 Agent 能力。
